# Screenshot Guide

Drop screenshots here with these exact filenames so the main README picks them up automatically.

Recommended size: ~1400px wide, PNG, dark browser chrome (or cropped to the UI).

| Filename | What to capture |
|---|---|
| `overview.png` | Main dashboard view — full table, tag bar visible, topbar with version and + New Device |
| `table.png` | Zoomed view of 4–6 rows showing status dots, IP links, platforms, versions (include one with "Update Available") and tags |
| `search.png` | Search bar with text typed in + a couple of tag pills active |
| `tags.png` | Tag editor modal open on a device showing current tags + "Existing tags (click to add)" pool |
| `side-panel.png` | Side panel slid in from the right with a device selected, Actions and Utilities both visible |
| `batch.png` | Multiple rows selected, blue batch bar visible at the top showing "N selected" |
| `compile-output.png` | Compile/logs modal mid-run with ANSI-colored output and DOWNLOAD LOGS / EDIT / CLOSE footer |
| `editor.png` | Ace YAML editor modal open with actual YAML visible |
| `inactive.png` | A couple of dimmed inactive rows at the bottom of the list |
| `archive.png` | Archived section expanded showing 2–3 archived devices |
| `new-device.png` | + New Device wizard modal open with the form fields visible |

## Tips

- Use a real config with interesting device names if possible
- Keep the browser address bar visible if it looks clean (optional)
- Dark mode only — the dashboard is dark-themed, so screenshots should match
- Crop whitespace generously — don't need the full browser window
